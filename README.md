# 📐 基于频域解析导向的多边形占用场重构方案总结

## 1. 数据采样与加载逻辑 (Data Pipeline)
为了强化模型对几何关键特征的感知，采用了一种**基于拓扑分析的混合采样策略**：

*   **拓扑清理（Topology Pre-processing）**：
    利用 `Shapely` 的 `unary_union` 操作将原始三角网格溶解（Dissolve）为纯净的多边形外轮廓，消除内部共享边。
*   **混合采样比例（Mixed Sampling Ratios）**：
    *   **Global (10%)**：全域随机采样，负责探测背景噪声，建立初步内外空间感。
    *   **Edge (50%)**：在线段上线性插值并辅以高斯抖动（Jitter），强制模型学习直线边界。
    *   **Corner (15%)**：在顶点处进行极小标准差的高斯采样，精准锁定多边形的锐角。
    *   **Inside (25%)**：利用 `matplotlib.path` 进行判定，确保多边形内部填充完整，无空洞。
*   **逐点权重（Per-point Weighting）**：
    为不同类型的采样点赋予不同的 Loss 权重（Edge=5.0, Corner=5.0, Inside/Global=1.0），引导网络优先攻克高频跳变区域。

## 2. 模型架构与数据流动 (Architecture & Data Flow)
模型摒弃了传统的纯空间映射，引入了**解析频率位置编码（Analytical Frequency Positional Encoding）**。

### 2.1 模型输入
1.  **查询坐标 (Coords)**：$[B, N, 2]$，空间中的连续坐标 $(x, y) \in [-1, 1]$。
2.  **频域资产 (Frequency Assets)**：$[B, 63, 32]$ 的实部与虚部矩阵，由 MAE Decoder 预测，代表物体的频谱特征。
3.  **语义嵌入 (Global Embedding)**：$[B, 384]$ 维向量，捕捉物体的宏观类别与全局形状特征。

### 2.2 核心逻辑与数据流向
1.  **频域参数化**：将实部与虚部转换为物理意义明确的**振幅 (Amplitude $A$)** 和 **相位 (Phase $\phi$)**。
2.  **解析调制（Analytical Modulation）**：
    不使用离散插值，而是利用傅里叶变换的解析本质，将坐标 $(x, y)$ 作为相位偏移器介入：
    $$\text{Response}_i = A_i \cdot \cos(2\pi(u_i x + v_i y) + \phi_i)$$
    这一步将 2D 坐标映射为一个由 **2016 条余弦波**组成的局部震荡响应向量。
3.  **特征融合 (Feature Fusion)**：
    将输入坐标 (2d)、解析频率响应 (2016d) 以及全局语义 (384d) 进行拼接，形成一个 **2402 维** 的高维特征表示。
4.  **主干网络 (Backbone)**：
    采用 **GELU MLP** 架构（维度：[512, 512, 256, 128]）。
    *   *注意*：弃用 SIREN，避免 $\sin(\cos(x))$ 的嵌套震荡，利用隐式频率响应直接驱动 MLP 进行边缘切割。

### 原有SIREN（区别）
第四步：送入 SIREN最后，我们将空间坐标和这组算好的响应向量拼接，喂给 SIREN：$$\mathbf{h}_{in} = [x, y, \mathbf{s}_{(x,y)}]$$$$\text{预测的概率 } OCF = \text{SIREN}(\mathbf{h}_{in})$$然后让这个预测的概率和真实的 0 或 1（该点到底在不在多边形内）计算 BCE Loss。总结：为什么这个设计极其精妙？在这个管线中，$(x, y)$ 永远留在空间域（时域），$\{A_i, \phi_i\}$ 永远留在频域。它们相遇的地方，就是余弦函数 $A_i \cos(2\pi(ux+vy)+\phi_i)$ 里面。这种做法极其优雅地避免了你所担心的“点变换后没有振幅”的问题，因为振幅 $A_i$ 是由 ViT 赋予的多边形的宏观形状属性，而 $(x,y)$ 只是去余弦波浪的特定位置“取样”而已

### 2.3 模型输出
*   **Logits**：$[B, N, 1]$，无界的原始预测值。在推理时通过 `Sigmoid` 转化为 $[0, 1]$ 的占用概率。

## 3. Loss 设计 (Optimization Objective)
采用**混合损失函数**，兼顾局部细节与全局拓扑：

1.  **加权 BCE + OHEM**：
    *   应用 Dataloader 传来的点权。
    *   引入 **在线硬样本挖掘 (OHEM)**，每一轮只计算前 80% 预测错误的点的损失，强制网络“攻坚”复杂的边界区域。
2.  **Log-Cosh Dice Loss**：
    *   作为全局形状约束，解决 BCE 在极度类别不平衡（如细长多边形）时的失效问题。
    *   使用 `Log-Cosh` 算子平滑 Dice 梯度，提高训练稳定性，防止边界破碎。

## 4. 训练与测试设计 (Training Strategy)

*   **并行策略**：采用 **DDP (Distributed Data Parallel)** 多卡分布式训练，加速收敛。
*   **优化算法**：
    *   **AdamW**：配合 `weight_decay=1e-6` 防止过拟合。
    *   **Cosine Annealing LR**：学习率从 `3e-4` 逐渐衰减至 `1e-6`，确保训练后期能够精准微调边缘。
*   **数据切分**：严格执行 **90% 训练 / 5% 验证 / 5% 测试** 的划分，并封存测试集索引。
*   **推理评估 (Inference)**：
    *   **分块推理 (Chunking)**：为了实现 $512 \times 512$ 以上的超高分辨率重构而不溢出显存，将全域网格点切分为 $64k$ 的块逐次推理。
